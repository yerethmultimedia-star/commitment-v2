# Engineering Prompts

## Purpose

Reusable prompts used during Product Development.

## Principles

- Reuse before creating new prompts.
- Keep prompts generic whenever possible.
- Do not encode slice‑specific knowledge into permanent prompts.
- Product delivery always takes precedence over process refinement.

## Stability Policy

The Prompt Library is considered stable.

Permanent prompts are created only after the same workflow has been successfully reused at least three times.

Changes to permanent prompts require objective technical evidence that the current prompt blocks delivery of a product capability.

Whenever possible, new prompt ideas should first be validated as examples before becoming permanent templates.

## Categories

- Vertical Slice
- Capability Review
- Technical Design Review
- Implementation Review
- Architecture Review
- Investigation
